Ping Command in Linux/Unix

Overview

The ping command is used to test the reachability of a host on a network. It sends ICMP echo request packets to the target host and waits for a response, helping diagnose network connectivity issues.

What Does Ping Command Do?

The ping command is used to:

  • Check if a host is reachable over the network.
  • Measure round-trip time for messages sent to the host.
  • Identify network latency and packet loss.

By default, ping continues to send packets until interrupted, but you can specify the number of packets to send.

Common Use Cases

  • Verify the availability of a server or website.
  • Diagnose network issues like high latency or dropped packets.
  • Measure the time it takes for data to travel to a host and back.

Options of Ping Command

Option Description
ping [hostname/IP] Sends packets to the specified host and displays results.
ping -c [count] [hostname/IP] Specifies the number of packets to send.
ping -i [interval] [hostname/IP] Sets the interval between packets in seconds.
ping -w [timeout] [hostname/IP] Specifies the total time to wait before stopping.
ping -s [size] [hostname/IP] Specifies the size of the packet to send.

Example

Here’s an example of using the ping command:

# Test connectivity to google.com
ping google.com

# Send 5 packets to 192.168.1.1
ping -c 5 192.168.1.1

# Set packet size to 128 bytes
ping -s 128 example.com

This example demonstrates how to test connectivity, limit the number of packets sent, and set a specific packet size.
