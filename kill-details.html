Kill Command in Linux/Unix

Overview

The kill command in Linux/Unix is used to terminate processes manually. It sends a specific signal to a process, instructing it to perform an action, such as exiting or reloading configuration files.

It is a vital tool for system administrators to control misbehaving or resource-heavy processes effectively.

What Does Kill Command Do?

The kill command communicates with processes via signals. By default, it sends the SIGTERM signal, politely asking a process to terminate. If the process doesn’t respond, the SIGKILL signal can be used to force termination.

Signals sent by the kill command include:

  • SIGTERM (15): Requests the process to terminate gracefully.
  • SIGKILL (9): Forces the process to terminate immediately.
  • SIGHUP (1): Reloads the process configuration without stopping it.

Common Use Cases

  • Terminate unresponsive applications or services.
  • Restart processes to apply new configurations.
  • Control processes consuming excessive resources.

The kill command is often used alongside commands like ps or top to identify and manage processes by their Process ID (PID).

Options of Kill Command

Option Description
-l Lists all signal names.
-s SIGNAL Specifies the signal to send (e.g., kill -s SIGKILL PID).
-n SIGNAL Sends a signal by its numeric value.
-L Displays signal names and their corresponding numbers.

Example

Here's an example of using the kill command:

# Find the process ID
ps aux | grep apache

# Kill the process
kill -9 12345

This example shows how to find and terminate a process using its PID.
